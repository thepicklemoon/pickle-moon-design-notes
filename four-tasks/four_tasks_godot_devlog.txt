ARCHITECTURE (locked):
  - Server is source of truth: Cloudflare Worker + D1 (Oceania), TS.
    Three-worker target dev/staging/prod. Godot client mirrors, polls,
    optimistic local writes w/ rollback.
  - Identity: user_id (person, forever) / pair_id (relationship) /
    pair_key (recovery hash). Self-writes only for v1.0; partner
    reactions deferred to v1.x.
  - HTTP envelope {ok,data}|{ok,error,message}; codes 200/400/404/409/500
    (+429 rate-limit only). No 401/403 (perm-denied folds to 404).
  - Sealing: LAZY walkback. Days sit unsealed; the morning claim
    (POST /users/:id/claim) walks back, seals the most recent unsealed
    accounted-for day, computes stamp/coins/streak then. `morning_payout_
    due_for` column dropped as redundant. (Eager-seal-at-completion
    considered + REJECTED session 26 — unnecessary over-engineering;
    lazy is sufficient.)
  - File naming: autoloads PascalCase (Backend/State/Identity); else
    snake_case; internal code always snake_case.

PHASES:
  - Phase 1 (backend), Phase 2 (calendar/tray/swipe/poll), Phase 3

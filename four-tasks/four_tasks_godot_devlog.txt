Top-level scene tree:
  Main (Node)
  ├── Backend (Autoload)        — server I/O, signals
  ├── State (Autoload)          — in-memory mirror of pair data
  ├── ThemeManager (Autoload)   — palette swap, theme_changed signal
  ├── Audio (Autoload, v1.x)    — slot ticks, coin sounds, stamp thump
  ├── Loader (Scene)            — identity check, pair-key resolve
  ├── Onboarding (Scene)        — 8-screen flow (session 9)
  ├── App (Scene)               — main two-panel UI
  └── DevKit (Scene, debug-only) — bug inbox, seed tools, palette
                                   derivation devtool (spec'd session 8)

Source of truth: server. State is a mirror.
Write flow: UI → State.set_*() → Backend.save() → server confirms
            → State emits signal → UI updates from signal.
Optimistic local updates with rollback on server reject.

Server architecture (locked session 2):
  - Cloudflare Worker (TypeScript) at
    four-tasks-api.thepicklemoon.workers.dev
  - D1 SQL database (Oceania region) bound as env.DB
  - Four tables: pairs, users, days, bug_reports
  - Pair-key migrations re-entrant + transaction-wrapped
  - Response envelope: uniform {ok, data} / {ok, error}
  - HTTP status codes: 200/400/404/409/500 (no 403)
  - Auth: caller named in URL; cross-user writes use ?caller=name

Endpoints implemented (session 3):

  - POST /resolve — derives pair-key from six values.

LOCKED DECISIONS, ALPHABETICAL (with session of lock):

    subscribers see their own past richness, not theft.

  Devlog timestamp convention (S8): "Last edit: YYYY-MM-DD
    HH:MM AWST" header line in devlog + todo. Bumped each

  Diary content validation (S4): trust client + 200-char cap,
    don't mirror wordlists server-side.

  Doubling vs flat reroll cost (S5 extended): doubling for
    escalating-significance features (coin name reroll), flat
    for casual features (MOTD reroll, locked at 90-110 coins).

  Founders pricing tier (S6.2): launch cohort gets permanent
    founders rate to year-2 cutoff. Distinct from founders
    flag (prototype-era free access).

  Gesture-dismissal pattern for reveals (S9): popup clears
    when user performs the action being taught. No "Got it"
    button. New reusable technique.

  Identity = pair-key v2 (S2; reopened S9): six-value tuple
    (name + username + icon × 2 users), hashed. Name immutable;
    username + icon mutable, trigger pair-key migration.
    Migrations re-entrant + transaction-wrapped. Section 4
    OPEN (Option A vs B data model).

  Library sharing as subscription mechanic (S6.2): subscription
    unlocks bilateral access to partner's purchased library.
    All packs bought with coins, owned permanently.

  MOTD font + UI fonts locked global (S7): not per-theme.
    Label backgrounds per pack; label typography stays global.

  MOTD reroll cost (S5 extended): flat 90-110 coins per press,
    no doubling. Supersedes prototype's doubling mechanic.

  Morning sequence as overlay (S5 extended): base state always
    calendar-flat; sequence is an overlay-on-top, not a boot
    mode. Same code path always.

  Morning sequence is private (S5 extended): partner panel
    reflects their yesterday as unstamped until they claim.
    Claim is the moment state becomes visible to partner.

  Onboarding solo-as-default (S9): no fork screen. Every user
    finishes onboarding solo. Pairing is a post-onboarding
    action via persistent "Add my partner" button on partner
    calendar.

  Pairing-by-typed-values (S9): joiner types name + username
    + icon, same as inviter's. Single /join_by_values endpoint

GOTCHAS (DEV ENVIRONMENT):

    use single quotes or backtick-escape.
  - Windows curl is Invoke-WebRequest alias — use curl.exe.
  - PowerShell mangles inline JSON — use --data-binary @file.
  - `wrangler dev` drifts to remote without --local flag.
  - PowerShell escape-quote corrupts SQL strings — use
    --file= references.

    Worker (~30min edge cert propagation). Browser works
    immediately.

    + `wrangler login` before assuming worse.
  - GitHub raw URL CDN has ~1-3min propagation lag after push.
  - web_fetch truncates around 2825 lines by default. Use
    explicit higher token limit for files larger than that.

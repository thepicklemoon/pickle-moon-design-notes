Top-level scene tree:
  Main (Node)
  ├── Backend (Autoload)        — server I/O, signals
  ├── State (Autoload)          — in-memory mirror of pair data
  ├── ThemeManager (Autoload)   — palette swap, theme_changed signal
  ├── Audio (Autoload, v1.x)    — slot ticks, coin sounds, stamp thump
  ├── Loader (Scene)            — identity check, pair-key resolve
  ├── Onboarding (Scene)        — 8-screen flow (session 9)
  ├── App (Scene)               — main two-panel UI
  └── DevKit (Scene, debug-only) — bug inbox, seed tools, palette
                                   derivation devtool (spec'd session 8)

Source of truth: server. State is a mirror.
Write flow: UI → State.set_*() → Backend.save() → server confirms
            → State emits signal → UI updates from signal.
Optimistic local updates with rollback on server reject.

Server architecture (locked session 2):
  - Cloudflare Worker (TypeScript) at
    four-tasks-api.thepicklemoon.workers.dev
  - D1 SQL database (Oceania region) bound as env.DB
  - Four tables: pairs, users, days, bug_reports
  - Pair-key migrations re-entrant + transaction-wrapped
  - Response envelope: uniform {ok, data} / {ok, error}
  - HTTP status codes: 200/400/404/409/500 (no 403)
  - Auth: caller named in URL; cross-user writes use ?caller=name

Endpoints implemented (session 3):

    use single quotes or backtick-escape.
  - Windows curl is Invoke-WebRequest alias — use curl.exe.
  - PowerShell mangles inline JSON — use --data-binary @file.
  - `wrangler dev` drifts to remote without --local flag.
  - PowerShell escape-quote corrupts SQL strings — use
    --file= references.

    Worker (~30min edge cert propagation). Browser works
    immediately.
  - Wrangler D1 import auth-error 10000: try `wrangler logout`
    + `wrangler login` before assuming worse.
  - GitHub raw URL CDN has ~1-3min propagation lag after push.
  - web_fetch truncates around 2825 lines by default. Use
    explicit higher token limit for files larger than that.
    (Devlog v1 hit this at session 9.)

# Four Tasks — Sticker Pack Brainstorm

Status: RUNNING DOC. Add to this as inspiration hits.

Purpose: catalogue ideas for theme leaders and their accompanying
sticker packs + background treatments. Implementation-agnostic —
this is a content brainstorm, not architecture. The "how" lives in
four_tasks_theme_design_notes.md.

Convention per entry:
  - Leader sticker name.
  - Companion stickers (pool members that fit the theme's vibe).
  - Background treatment idea (the calendar grid / unmarked cells /
    overall vibe when this is leader).
  - Notes — optional. Free-form thoughts on silhouette, palette
    direction, tier ambition, etc.

Tier targets (from theme doc):
  - TIER 1 = full theme: leader + companions + background + flourish.
  - TIER 2 = partial: some theme files, not all.
  - TIER 3 = palette-only: just sticker + palette.tres.

Onboarding shows TIER 1 only (the showpieces). Catalogue grows over time.

NOTES ON USING THIS DOC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- This doc is brainstorm. Nothing here is locked. Cross out, rewrite,
  delete, reorganise freely.
- When a pack feels ready to paint, it graduates to a TIER plan
  with paint order and ships against tile 4.D (sticker art).
- The "Background covers day tiles" frog idea is the kind of detail
  worth flagging — at implementation time it may need a special
  rendering treatment beyond the standard cell_unmarked.png slot.
  Capture these inline as they occur.
- Companion stickers don't have to be themed; they can be palette-
  only (TIER 3) and still belong to the pack vibe. The theme's
  "feel" comes from the leader + the curated companion list, not
  from every sticker shipping with theme art.
